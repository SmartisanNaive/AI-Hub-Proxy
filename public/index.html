00 · 简介

它解决了什么问题

当你身处无法直接访问 OpenAI、Claude、Gemini、Groq 等服务的地区时，AI Hub Proxy 可以部署在一台合规 VPS 上，作为中间代理。客户端只需连接一个中转端点，服务端统一处理鉴权与转发，从而绕过地域限制，让受限地区也能顺畅使用多家 AI 服务。 

# 以前：客户端需要直接访问多个被限制的域名
curl https://api.openai.com/v1/chat/completions
curl https://api.anthropic.com/v1/messages
curl https://api.groq.com/openai/v1/chat/completions

# 现在：统一走中转代理（将 <your-domain> 替换为你的实际域名）
curl https://<your-domain>/openai/v1/chat/completions
curl https://<your-domain>/claude/v1/messages
curl https://<your-domain>/groq/openai/v1/chat/completions

01 · 能力

核心特性

  1. 路径转发 Prefix-based routing

    每个服务对应一个独立路径前缀，例如 /openai、/claude、/gemini。前缀后的完整路径原样透传到目标服务，无需逐一适配。

  2. 请求头过滤 Minimal header pass-through

    默认只透传 accept、content-type、authorization 以及自定义 x-* 头，避免把无关的主机信息、Cookie 或内部标记泄露给上游。

  3. 安全响应头 Security headers by default

    自动附加 X-Content-Type-Options: nosniff、X-Frame-Options: DENY、Referrer-Policy: no-referrer 等基础安全头。

  4. 健康检查 Health endpoint

    内置 /health 接口，Docker Compose 可据此做健康检查；也便于负载均衡器或监控脚本判断服务状态。

02 · 设计

设计原则

  1. 01
    透明转发 不修改请求体与响应体，让上游 API 的行为保持原样。
  2. 02
    最小权限 只透传必要的头，减少攻击面和意外信息泄露。
  3. 03
    无状态 不缓存、不存储密钥，每个请求独立处理，便于水平扩展。
  4. 04
    单一容器 一个镜像、一份 Compose 文件即可运行，降低运维复杂度。

04 · 部署

项目部署

将本项目部署到一台可访问目标 AI 服务的 VPS 上，然后把域名指向该服务器，即可作为中转端点对外提供服务。以下命令中的 <your-domain> 请替换为你的实际域名。 

# 1. 克隆项目并进入目录
cd ai-hub-proxy

# 2. 启动服务（默认映射到宿主机 9000 端口）
docker compose up -d

# 3. 配置域名解析到 VPS，例如 your-domain.com
# 4. 在服务器前配置 HTTPS 反代（如 Nginx / Caddy）

# 5. 测试健康检查
curl https://<your-domain>/health

# 6. 调用 OpenAI API
curl https://<your-domain>/openai/v1/chat/completions \\
  -H "Authorization: Bearer $OPENAI_API_KEY" \\
  -H "Content-Type: application/json" \\
  -d '{"model":"gpt-4o","messages":[{"role":"user","content":"hello"}]}'

05 · 问答

常见问题

这个代理会保存我的 API Key 吗？
不会。请求头中的 authorization 会原样透传给上游服务，服务本身不读取、不记录、不缓存任何密钥。
如何添加新的上游服务？
编辑 main.ts 中的 apiMapping 对象，添加新的路径前缀与目标域名，然后重新构建镜像即可。
可以修改监听端口吗？
可以。修改 docker-compose.yml 中的端口映射，例如 9000:8080 表示把容器内的 8080 映射到宿主机的 9000。
这个页面也是代理提供的吗？
是的。访问根路径 / 或 /index.html 时，代理会返回这个静态页面，用于说明项目用途。
